# Outline Patterns

Choose the structure that best matches the task.

## Pattern 1: Taxonomy survey

1. Introduction and scope
1.1 Problem framing
1.2 Why this survey is needed
1.3 Scope and exclusions
2. Background and core concepts
2.1 Terminology
2.2 System setting
2.3 Key assumptions
3. Taxonomy
3.1 Category A
3.2 Category B
3.3 Category C
4. Representative methods by category
4.1 Storage or representation
4.2 Update policy
4.3 Retrieval or usage
5. Evaluation settings
5.1 Benchmarks
5.2 Metrics
5.3 Failure modes
6. Limitations
7. Open problems

## Pattern 2: Method comparison

1. Introduction and motivation
2. Comparison criteria
2.1 Representation
2.2 Update
2.3 Retrieval
2.4 Cost and controllability
3. Method family A
3.1 Core mechanism
3.2 Representative systems
3.3 Strengths and weaknesses
4. Method family B
5. Method family C
6. Empirical comparison
6.1 Benchmarks
6.2 Results
6.3 Failure analysis
7. Practical recommendations

## Pattern 3: Research landscape

1. Background
2. Key threads in the literature
2.1 Thread A
2.2 Thread B
2.3 Thread C
3. Main technical tensions
3.1 Tension A
3.2 Tension B
4. Benchmarks and datasets
5. Failure modes and limitations
6. Emerging directions

## Subsection quality checklist

- Does each subsection have a distinct comparison purpose?
- Can each subsection be supported by at least `2-3` relevant sources?
- Does the subsection title encode a real distinction rather than a vague theme?
- Is the outline deep enough that a writer could draft section by section without inventing new structure on the fly?

## Reminder

Use the pattern as a scaffold only. Real subsection boundaries should come from clustered reading notes and evidence maps.
